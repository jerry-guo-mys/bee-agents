## 错误日志模板

### 记录格式
```yaml
日期: 2026-02-17
时间: 14:30
会话ID: [如果有]

错误分类:
  类型: [意图误解/工具误用/路径错误/输出不当/其他]
  严重程度: [低/中/高/严重]

错误描述:
  场景: [用户请求内容]
  我的操作: [我做了什么]
  错误结果: [发生了什么]
  用户反馈: [用户的纠正/反应]

根因分析:
  表面原因: [直接的错误]
  深层原因: [为什么会犯这个错误]
  系统性问题: [是流程缺陷还是执行问题]

解决方案:
  立即修正: [当时怎么解决的]
  长期预防: [如何避免再次发生]
  方案更新: [需要更新哪个文档]

经验教训:
  - [要点1]
  - [要点2]
```

### 示例条目
```yaml
日期: 2026-02-17
时间: 10:15

错误分类:
  类型: 工具误用
  严重程度: 中

错误描述:
  场景: 用户说"计算 2+2"
  我的操作: 使用了 glob("*")
  错误结果: 列出目录而不是计算
  用户反馈: "我要的是计算结果，不是文件列表"

根因分析:
  表面原因: 选择了错误的工具
  深层原因: 没有建立"计算→bash"的直接映射
  系统性问题: 意图-工具映射表缺失该条目

解决方案:
  立即修正: 改用 bash("echo '2+2' | bc")
  长期预防: 更新工具选择速查表，添加计算任务映射
  方案更新: ai-improvement-plan.md, ai-quick-reference.md

经验教训:
  - 计算类任务必须使用shell命令
  - 常见任务需要建立直接映射
  - 不确定时先确认再执行
```
